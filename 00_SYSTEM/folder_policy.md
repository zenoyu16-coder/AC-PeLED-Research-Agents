# Folder Policy

## 根目录

只放项目入口文件。不要堆放 PDF、图片、CSV、导出图、压缩包或临时文件。

## `00_SYSTEM/`

**放置：** 项目规则、标准 workflow、目录政策。  
**不放：** 具体论文、实验记录、agent 输出。

## `01_PAPERS/`

**放置：** 文献模板、paper card 结构化信息、comparison table。  
**不放：** 杂乱截图、未命名 PDF、聊天记录或无法追溯来源的摘录。

## `02_PROTOCOLS/`

**放置：** 可复用、可填写、可审核的实验 protocol。  
**不放：** 单次实验结果、未验证的仪器说明或从论文复制但未适配的步骤。

## `03_DATA_ANALYSIS/`

**放置：** CSV 格式标准与可复现分析脚本。  
**不放：** 原始数据、手工修改后的图、二进制仪器文件或环境密钥。

## `04_AGENTS/`

**放置：** agent prompts、职责边界与输出格式。  
**不放：** 模型权重、API key、复杂 agent framework 或机器人控制代码。

## `05_REPORTS/`

**放置：** 最终输出模板。  
**不放：** 未审核草稿、原始数据副本或临时图。

## 命名与引用

- 文件名使用稳定、可读的英文 snake_case。
- 报告中引用数据时记录相对路径或受控存储 URI。
- 删除或重命名模板前检查所有引用。
- 临时分析应在仓库外完成，确认需要长期复用后再更新标准脚本。
